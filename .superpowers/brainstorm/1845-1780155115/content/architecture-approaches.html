架构选型：三个决策

为未来扩展打好基础

📌 决策一：路由方案

A

React Router v7

正式路由方案，URL 即状态
支持懒加载、深层链接

未来加页面不改架构
4个Tab稍重
B

状态驱动 Tab（推荐）

useState 控制当前 Tab
纯客户端，零依赖

极简、够用
URL不可分享

📌 决策二：状态管理

A

Zustand

轻量状态库，按 module 拆分 store
自带 subscribe + selector 优化

优雅的解耦
多一个依赖
B

Custom Hooks（推荐）

useTodos / useHealthData / useSettings
useLocalStorage 封装底层

零依赖、直观
跨模块通信需提升

📌 决策三：推荐组合

状态 Tab + Custom Hooks + localStorage

理由：4 个 Tab 用路由大材小用；Zustand 很好但你的数据全部 sync 到 localStorage，Custom Hooks 封装同样干净且少一个依赖。
